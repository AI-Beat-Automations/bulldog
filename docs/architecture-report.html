Overview

bulldog (a.k.a. "Bulldog Chat") is an embeddable AI chat widget plus a single-admin control panel. Anonymous visitors chat from an iframe widget backed by an OpenRouter LLM; the admin edits the assistant's behaviour (a versioned System Prompt), reviews persisted conversations, and browses ingested phone-call records (Retell → n8n) with public recording playback.

Stack

  • Framework — Next.js 16.2.9 (App Router, React Server Components), React 19.2.4. Node runtime on data routes.
  • Database — Postgres (Neon) via Drizzle ORM 0.45 on the postgres (postgres.js) driver. Migrations with drizzle-kit.
  • Auth — next-auth v5 (beta) Credentials provider, JWT sessions, single hard-coded admin.
  • AI — Vercel AI SDK (ai v6, @ai-sdk/react) with the OpenRouter provider; default model anthropic/claude-haiku-4.5.
  • UI — Tailwind v4 (CSS-first), shadcn primitives over the unified radix-ui package, lucide icons, custom light/dark theme (no next-themes).
  • Infra — Upstash Redis (rate limiting), n8n webhooks (inbound call ingestion + outbound availability lookup). Deployed on Vercel.

API Endpoints

4 HTTP route handlers plus 3 server actions and 1 LLM tool. No zod — handlers do manual type-guarding. Three distinct auth strategies coexist: session (JWT cookie) for the admin, public origin-allowlist + IP rate-limit for chat, and a bearer shared secret for the n8n webhook.

Rendering & theme notes

  • Styling — Tailwind v4 CSS-first (@import "tailwindcss" + tw-animate-css + shadcn/tailwind.css); tokens in @theme inline; dark via @custom-variant dark; neutral base with an amber primary; --radius: 0.625rem. shadcn style radix-nova, RSC on, lucide icons.
  • Theme system — deliberately no next-themes and no React Context. theme-provider.tsx is an external module store read through useSyncExternalStore: state lives in module scope, subscribe() listens to matchMedia + storage (cross-tab); setTheme() writes localStorage, toggles .dark + colorScheme on <html>. A blocking themeInitScript IIFE in the root body sets .dark pre-hydration (anti-FOUC).

Auth, Roles & Middleware

Mechanism

  • next-auth v5 (beta), Credentials provider only, JWT sessions (session.strategy:"jwt", lib/auth.ts:39). No DB adapter, no OAuth.
  • authorize compares submitted email/password to AUTH_EMAIL/AUTH_PASSWORD (constant-time safeEq) and returns a single static identity { id:"admin", email } or null (lib/auth.ts:25-35).
  • Edge-safe — lib/auth.ts avoids node:crypto (uses TextEncoder-based compare) so the same module is importable by both the edge middleware (proxy.ts) and server. No separate auth.config.ts split. (Contrast: lib/webhook-auth.ts uses node:crypto.timingSafeEqual — it only runs in a Node route.)
  • Login at /login; sign-out via inline signOut action in (admin)/layout.tsx:45.

Middleware & guards

  • proxy.ts (Next.js 16 middleware, renamed from middleware.ts) — export const proxy = auth(...). Allow-lists /login, /widget, /audio-call, /api/chat, /api/auth, /api/webhooks; everything else requires a session (!req.auth?.user → redirect /login). Matcher excludes _next/*, favicon.ico, widget.v1.js, .svg. The /admin gate is thus enforced twice (middleware + layout).
  • CSP / clickjacking — next.config.ts sets Content-Security-Policy: frame-ancestors <ALLOWED_ORIGINS> only on /widget (fallback 'none'). Intentionally no X-Frame-Options: DENY (would break the embed). frame-ancestors is the primary embed gate.
  • CORS (lib/cors.ts) — origin allowlist = ALLOWED_ORIGINS + AUTH_URL self-origin; missing Origin is allowed. Enforced on /api/chat only (403). Exposes X-Conversation-Id. Defense-in-depth behind frame-ancestors.
  • Webhook bearer (lib/webhook-auth.ts) — verifyN8nSecret: 500 if secret unset, 401 if missing/invalid, constant-time compare.
  • Rate limit (lib/rate-limit.ts) — Upstash Redis sliding window 20 req / 10 min / IP (prefix bulldog-chat), keyed off first x-forwarded-for hop. Degrades open (allows all) if Upstash env is absent.
Two accepted soft-spots: (1) chat source is client-supplied and falsifiable (degrades listing reliability, not security — ADR-0002); (2) rate limiting silently no-ops if Upstash env vars are unset.
